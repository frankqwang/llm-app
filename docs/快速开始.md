# 快速开始

## 环境准备

- Android Studio Ladybug+（或 Hedgehog + AGP 8.7+）
- **JDK 21 from Adoptium / Temurin** — LiteRT-LM 0.11.0 使用 Java 21 类文件。项目配置了 Gradle Daemon Toolchain，首次构建时会自动发现或自动下载 Temurin 21。也可手动安装：
  - Windows: `winget install EclipseAdoptium.Temurin.21.JDK`（或从 [Adoptium 官网](https://adoptium.net/temurin/releases/?version=21) 下载 `.msi`）
  - macOS: `brew install --cask temurin@21`
  - Linux: 发行版包管理器或 Adoptium 压缩包
- Android SDK Platform 35 + Build-Tools 35（Android Studio 会自动安装）
- 测试设备：Android 12+（minSdk 31），运行 Gemma 4 E2B-IT 建议 8GB 以上内存
- 接受 Gemma 4 许可的 Hugging Face 账号：https://huggingface.co/litert-community/gemma-4-E2B-it-litert-lm

## 侧载运行（推荐方式）

此方式跳过 HuggingFace OAuth，直接本地导入模型。

### 步骤 1 — 在电脑上下载模型

模型需要授权，因此必须先用已接受 Gemma 4 条款的 HF 账号登录。

```powershell
# PowerShell，先执行 pip install -U huggingface_hub 和 hf auth login
# 注意：huggingface-cli 已弃用，新二进制名为 hf
hf download litert-community/gemma-4-E2B-it-litert-lm `
  gemma-4-E2B-it.litertlm `
  --local-dir C:\dev\llm-app\models
```

下载后得到 `C:\dev\llm-app\models\gemma-4-E2B-it.litertlm`（约 2.6 GB）。`models/` 目录已被 gitignore。

该 HF 仓库共有 4 个文件（10.9 GB），**只需下载通用的 `.litertlm`**。跳过以下文件：
- `gemma-4-E2B-it-web.task` — MediaPipe 运行时，非 LiteRT-LM
- `gemma-4-E2B-it_qualcomm_qcs8275.litertlm` — 高通 QCS8275 NPU 专用版
- `gemma-4-E2B-it_qualcomm_sm8750.litertlm` — 骁龙 8 Elite NPU 专用版

高通专用 NPU 版无法在联发科（天玑）或 Tensor SoC 上运行。

### 步骤 2 — 构建并安装 APK

```powershell
cd C:\dev\llm-app\android
.\gradlew.bat assembleDebug
adb install -r app\build\outputs\apk\debug\app-debug.apk
```

无需填写 HF OAuth 占位符——这些仅用于应用内下载流程，而我们在侧载时已经绕过。只要不在应用内尝试登录，构建就会成功。

### 步骤 3 — 推送模型到手机

```powershell
adb push C:\dev\llm-app\models\gemma-4-E2B-it.litertlm /sdcard/Download/
```

`/sdcard/Download` 是最方便的位置，因为系统文件选择器会将其显示在"下载"下。

### 步骤 4 — 在应用内导入模型

1. 在手机上打开应用。
2. 打开侧边栏 / "Models" 页面。
3. 点击右下角的 **+** 浮动按钮。
4. 选择 **From local model file**。
5. 在系统文件选择器中，导航到 **Downloads** → `gemma-4-E2B-it.litertlm`。
6. **关键步骤** — 在导入对话框中点击确认前，打开以下开关：
   - **Support speculative decoding** — 启用 MTP。若关闭，SPECULATIVE_DECODING 能力不会附加到模型，聊天配置也不会显示运行时开关，v0.11.0 的性能提升将无法生效。
   - **Support thinking** — 仅在需要思考模式时开启（默认关闭对原始解码基准测试无影响）
   - **Support image** — 建议开启。VlogPilot 不依赖此标志（运行时直接向引擎请求视觉能力），但 gallery 的其他视觉任务（如 LLM Ask Image）会按此过滤。
   - **Support audio** — 仅在需要音频输入时开启
   - **Compatible accelerators** — 确保 GPU 已选中
7. 点击确认。应用会将文件复制到自己的存储空间 `/sdcard/Android/data/com.google.aiedge.gallery/files/__imports/gemma-4-E2B-it.litertlm` 并注册。
8. 显示 "Model imported successfully" 后，模型会出现在 "Imported Models" 下，并可在 **AI Chat** 任务中使用。

步骤 7 完成后，可以执行 `adb shell rm /sdcard/Download/gemma-4-E2B-it.litertlm` 回收手机上的 2.6 GB 空间（应用已在 `__imports/` 下保留自己的副本）。

### 步骤 5 — 性能优化（vivo X200 Pro / 天玑 9400）

首次启动时默认可能运行在 CPU 上。在聊天配置中（聊天界面的齿轮图标）：

- **Accelerator**: GPU（GPU 初始化失败时回退到 CPU）
- **Multi-token prediction (MTP)**: ON — v0.11.0 的核心功能，最大性能提升来源
- **Thinking mode**: OFF（会注入思考 Token，增加单条响应的壁钟时间）

天玑 9400 + GPU + MTP 预期可达 40–60 tok/s 解码速度（基于 Google 公布的同 SoC 级别 S26 Ultra 基准推算）。

## 其他方式（参考）

### 使用官方 APK 快速验证

- Play Store: `com.google.ai.edge.gallery`
- 或从 GitHub Release 下载：https://github.com/google-ai-edge/gallery/releases/latest

### 从源码构建并启用 Hugging Face OAuth

仅在需要应用内 HF 登录 + 自动下载流程时使用。

1. 注册 OAuth 应用：https://huggingface.co/settings/applications/new
   - Redirect URI: 使用你控制的 scheme，如 `com.example.gemmaapp://oauthredirect`
   - Scopes: `read-repos`
2. 替换以下文件中的占位符：
   - `android/app/src/main/java/com/google/ai/edge/gallery/common/ProjectConfig.kt`（`clientId`、`redirectUri`）
   - `android/app/build.gradle.kts`（`manifestPlaceholders["appAuthRedirectScheme"]`）
3. 在 Android Studio 中打开 `android/` 目录，同步 Gradle，点击 Run。

## 故障排查

### 运行流水线时查看相关 logcat

```powershell
adb logcat -c                                                        # 清空
adb logcat AGLlmChatModelHelper:D LiteRtLm:D Engine:D AndroidRuntime:E *:S
```

关注以下信息：
- `Preferred backend: GPU` — 确认已选择 GPU（显示 CPU 表示运行时回退）
- `Speculative decoding enabled: true` — 确认补丁 + 导入开关生效，MTP 已对本会话启用
- 任何 `Failed to initialize` / `falling back to CPU` 行 — 通常表示 GPU 后端拒绝了模型或设备

### 解码速度仍是单位数 tok/s

按可能性排序：

1. **导入时未启用 MTP 开关。** 重新导入模型，在导入对话框中打开 "Support speculative decoding"。
2. **聊天配置中关闭了运行时 MTP。** 打开聊天 → 齿轮图标 → 查看 "Enable speculative decoding"。如果开关完全消失，说明 `ModelPageAppBar.kt` 中的补丁未生效或导入时的能力未持久化——请重新检查两者。
3. **后端静默回退到 CPU。** Logcat 会显示 `Preferred backend: CPU`。打开聊天配置 → Accelerator → GPU。
4. **思考模式开启且回答冗长。** 不会改变 tok/s，但会增加单条响应的壁钟时间。基准测试时关闭。
5. **首条消息预热。** GPU 着色器编译会惰性发生在首次推理时（约 5–15 秒）。请在第二条消息后测量，而非第一条。

### "+" 导入按钮不显示

请从侧边栏 → Models 进入。FAB 位于该页面，而非首页的任务卡片上。

### 导入对话框拒绝文件

执行 `adb logcat -s ModelImportDialog:*` 查看原因。常见原因：文件名包含空格或非 ASCII 字符，或存储权限被拒绝。
