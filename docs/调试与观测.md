# 调试与观测

vivo OriginOS（及类似 OEM 系统）的 `HWUI`/`SurfaceComposerClient` 会以极快速度填充 logcat，约 7 分钟内覆盖 4 MB 的环形缓冲区，淹没 `Log.d` 条目。为解决这个问题，每次运行都会写入磁盘：

## 文件结构（位于 `/data/data/com.google.aiedge.gallery/files/`）

| 文件 | 内容 |
|---|---|
| `_state.txt` | 单行：`<unix_millis>\t<stage>\t<detail>` — 当前流水线状态 |
| `_state_history.jsonl` | 追加式记录每个 PipelineProgress 事件的历史 |
| `_state_fatal.txt` | `run()` 在逐事件 catch 之上抛出时的堆栈跟踪 |
| `agent_log/agent.jsonl` | 每次 LLM 调用的耗时 + 结果（`ok` / `timeout` / `error:Foo`）；迭代调用前缀为 `iter_v<N>/` |
| `decisions/<eventId>/*.json` | EventMemory / AudienceBrief / DirectorBrief / Timeline / Critique / perf / user_brief |
| `decisions/<eventId>/_history.json` | IterationHistory：每个版本的 mp4 + 反馈文本 + 变更摘要 |
| `decisions/<eventId>/iter_<N>/feedback_input.txt` | 第 N 次迭代的用户原始反馈文本 |
| `decisions/<eventId>/iter_<N>/feedback_parsed.json` | IntentParserAgent 的结构化解读 |
| `decisions/<eventId>/iter_<N>/timeline.json` | 本次迭代用于渲染的时间线快照 |
| `decisions/<eventId>/iter_<N>/perf.json` | 每次迭代的 parseMs / editorMs / renderMs / totalMs |
| `_pending_iteration/<eventId>.json` | 等待 Worker 处理的暂存反馈（冷启动时自动恢复） |
| `user_curation_requests/<requestId>.json` | 用户的原始 UserCurationRequest（用于重新提交时的缓存命中） |
| `perception_cache/<assetId>.json` | 逐资产 Perception（轻量信号 + VLM 标签） |
| `event_scout_cache/<eventId>.json` | EventScout VLM 信号（故事弧 / 人物 / 节奏） |
| `candidates/<eventId>.mp4` | 最新渲染的 vlog（始终是最新版本） |
| `candidates/archive/<eventId>__<timestamp>.mp4` | 旧版本（"上一版"箭头播放这些） |

## 读取方式

```powershell
adb shell run-as com.google.aiedge.gallery cat files/_state.txt
adb shell run-as com.google.aiedge.gallery cat files/_state_history.jsonl
adb shell run-as com.google.aiedge.gallery cat files/agent_log/agent.jsonl
```

或实时流式查看（非 Windows sleep-polling）：

```bash
while true; do
  adb shell run-as com.google.aiedge.gallery cat files/_state.txt
  echo
  sleep 2
done
```

## 扩展 logcat 缓冲区

如果你拥有不受限制的 logcat（可用 `adb logcat -d -t 5` 验证是否返回任何带有 `PipelineOrchestrator` 标签的内容），先扩大缓冲区以保留我们的 `Log.d` 条目：

```bash
adb logcat -G 16M
adb logcat -v threadtime PipelineOrchestrator:D AgentRuntime:D EditorAgent:D *:S
```
