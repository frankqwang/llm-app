# 与上游的差异及未来清理

## 与上游 gallery 的本地修改

- **`customtasks/vlogpilot/` 下的 VlogPilot 模块** 是本分支原创（上游不存在）。
- `android/gradle/libs.versions.toml`：将 `litertlm` 从 `0.10.0` 升级到 `0.11.0`（2026-05-05 发布）。v0.11.0 为 Gemma 4 启用多 Token 预测（MTP），在移动 GPU 上解码速度提升 2 倍以上。
- `android/app/.../ui/llmchat/LlmChatModelHelper.kt`、`ui/common/ModelPageAppBar.kt`：**Bug 修复** — 上游硬编码了本地 `supportsSpeculativeDecoding = false` 桩，导致导入对话框中隐藏了 MTP 开关，并在模型声明能力时静默禁用 MTP。已修复为读取 `model.capabilityToTaskTypes`。
- `ui/navigation/GalleryNavGraph.kt`：应用启动直接进入 VlogPilot（`ROUTE_VLOGPILOT`）；gallery 侧边栏仍可到达，但不再是首页。

## 未来可清理的代码（不阻塞 VlogPilot 运行）

本分支仍携带上游 gallery 中 VlogPilot 不使用的代码路径。它们会增加 APK 体积和构建时间，但目前没有损坏；清理属于 APK 体积优化，而非正确性需求。

- `customtasks/{agentchat,tinygarden,mobileactions,examplecustomtask}/` — gallery 自带的其他任务，在我们的应用中已废弃
- `ui/{llmaskimage,llmpromptlab,auth}/` — 非 VlogPilot UI
- `data/SkillAllowlist.kt` + 仓库根目录的 `skills/` 目录 — Skills 系统
- `runtime/aicore/AICoreModelHelper.kt` — AICore（通过 Android System Intelligence 的 Gemini Nano）；我们仅使用 `litert_lm`
- HF OAuth 流程（`common/ProjectConfig.kt` 中的 OAuth 部分、`worker/DownloadWorker.kt` 的 HF 部分）— VlogPilot 复用 gallery 的本地文件导入，而非 OAuth 下载
- `app/build.gradle.kts` 中可移除的依赖：`libs.tflite*`、`libs.camerax.*`、`libs.firebase.*`、`libs.openid.appauth`、`libs.play.services.oss.licenses`、`google-services` 插件

> **注意**：在 VlogPilot 功能锁定前不要执行这些清理。每次移除都需要重新构建 + 冒烟测试。
